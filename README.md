# Backend-Self-Learning 

This repository is my personal backend engineering lab.  
It contains practical experiments, implementations, and notes built from scratch as I explore and learn backend development in depth.

Everything here is created independently to test and understand concepts — from HTTP protocol behavior, headers, and CORS, to validations, rate limiting, database relations, authentication, and other backend tools and patterns.

---

## Purpose 🤔

The main goal of this repository is to:

- Break down backend concepts beyond surface-level usage  
- Experiment with protocol-level behavior (HTTP, CORS, headers)  
- Explore practical backend tools and patterns (validations, rate limiting, email, auth, etc.)  
- Understand how browsers, servers, and databases interact  
- Build a long-term intuition for scalable and reliable backend design  

---

## Current Topics Covered

- **HTTP & Headers**: Accept, Content-Type, status codes, caching, compression  
- **CORS**: Simple vs non-simple requests, preflight mechanics, header enforcement  
- **Authentication & Tokens**: Authorization header, refresh token flows, basic auth patterns  
- **Data & Relations**: Handling database relations, reference patterns, query experiments  
- **Email Handling**: Nodemailer, SMTP setup, sending strategies  
- **Other Backend Tools**: Validations, middleware patterns, rate limiting experiments  

> This is an evolving lab — every new backend concept I learn is added here with examples and notes.

---

## Approach 

- Minimal, focused implementations for clarity
